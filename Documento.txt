Documentação do Sistema Gestro-OS


1. Introdução 
    O Gestor-OS é um sistema desenvolvido em Django para gerenciamento de Ordens de Serviço (OS).
    Ele foi criado para organizar o fluxo de abertura, acompanhamento e execução de ordens de serviço, permitindo maior controle e               eficiência nos processos internos.

  1.1 Objetivos
    -> Centralizar as Informações das OS
    -> Permitir a abertura e consulta de OS de forma Pratica
    -> Controlar a execução de atividades por colaboradores
    ->  


2. Tecnologias Utilizadas
  2.1 Backand
    -> Django (Python)
  2.2 Frontand
    -> HTML5
    -> TailwindCSS
    -> JavaScript (AJAX)
  2.3 Banco de Dados
    -> SQLite (em desenvolvimento) / PostgreSQL (produção sugerida)

3. Instalação e Configuração 
  3.1 Pré-requisitos
    -> Python 3.10+
    -> Pip
    -> Virtualenv ( opcional, mas recomendado)

  3.2 Passos
    3.2.1 Clonar repositório 
        -> git clone https://github.com/EricMoschen/Gestor-Os.git
    3.2.2 Entrar no diretório
        -> cd GESTOR-OS
    3.2.3 Criar ambiente virtual
        -> python -m venv venv
           source venv/bin/active  # Linux/Mac
           vevn\Scripts\activate   # Windows
    3.2.4 Instalar dependências
        -> pip install -r requirements.txt
    3.2.5 Migrar Bancos 
        -> python manage.py migrate
    3.2.6 Criar superusuário
        -> python manage.py createsuperuser
    3.2.7 Rodar servidor 
        -> python manage.py runserver


4. Estrutura do Sistema
  
    4.1 Apps Principais
      4.1.1 core
        -> Responsável pela configuração do sistema
      4.1.2 usuario
        -> Responsável pela validação do usuario 
      4.1.3 -> menuos
        -> Responsável pela abertura e gerenciamento das Os

    4.2 Models Principais

      4.2.1 CentroDeCusto
        -> Armazena os dados dos Centros de Custos cadastrados
      4.2.2 Cliente
        -> Armazena os dados dos Clientes cadastarados
      4.2.3 MotivoIntervencao
        ->  Armazena os dados dos motivos de intervenção cadastarados
      4.2.4 AberturaOS
        -> Armazena os dados das Ordens de serviço abertas
      4.2.5 Colaborador
        -> Armazena os dados dos Colaborador cadastarados
      4.2.6 RegistroInicioOS
        -> Armazena os registro de inicio e fim das horas dos colaboradores nas Ordem de Servico
      4.2.7 OrdemServico
        -> Armazenam as definiçoes de acessos para cadas usuario
